## 五、PID 温度控制测试

| 序号 | 测试项 | 预期结果 |
|------|--------|----------|
| 5.1 | 初始状态（-18 = 目标） | PID 输出接近 0，压缩机关闭 |
| 5.2 | 自然升温 | 压缩机关闭时温度每秒 +0.1，偏离目标后 PID 输出增大 |
| 5.3 | 压缩机启动 | `pid_output > 30` 时红色 LED（GPIO 18）亮，串口打印 `Compressor ON` |
| 5.4 | 制冷降温 | 压缩机开启后温度每秒 -0.5 |
| 5.5 | 压缩机停止 | `pid_output < 10` 时红色 LED 灭，串口打印 `Compressor OFF` |
| 5.6 | 温度上下限 | 温度范围 [-30, +10] |

### 关于 OLED 上 `Comp` 的含义

`Comp` 即 **Compressor（压缩机）**，OLED 主界面显示 `Comp: ON` 或 `Comp: OFF`，表示制冷压缩机的工作状态：

- **ON**：PID 输出 `pid_output > 30.0` 时启动，红色 LED（GPIO 18）同步亮起。当前温度高于目标温度，需要制冷。
- **OFF**：PID 输出 `pid_output < 10.0` 时停止，红色 LED 同步熄灭。温度已接近或达到目标，无需制冷。

此 30/10 的滞回区间防止压缩机在阈值附近频繁启停。

## 七、安全检查与故障模式测试

| 序号 | 测试场景 | 预期结果 |
|------|----------|----------|
| 7.1 | 温度超过 +10 | 进入 ERROR 模式，蜂鸣器（GPIO 21）间歇响，压缩机和除霜关闭，串口打印 `ERROR: Temperature too high!` |
| 7.2 | 温度低于 -30 | 进入 ERROR 模式，同上，串口打印 `ERROR: Temperature too low!` |
| 7.3 | ERROR 模式下 OLED | 主界面 Mode 栏显示 `ERROR` |
| 7.4 | 从 ERROR 恢复 | 菜单 Reset 或手动切换模式，系统恢复正常 |

> 触发故障的快捷方法：反复调高 Set Temp 至 10，等待压缩机停机后温度自然上升超过 10；或将 Set Temp 调到 -30，让制冷将温度拉至 -30 以下。

## 建议测试顺序

1. **一** — 启动确认
2. **四** — 串口键盘（最稳定的控制方式，不受硬件按钮状态影响）
3. **三** — 硬件按钮（确认按钮功能）
4. **二** — OLED 显示（确认两种界面切换正常）
5. **五** — PID 控制（核心功能）
6. **六** — 系统模式切换
7. **七、八、九、十** — 边界条件与辅助功能
